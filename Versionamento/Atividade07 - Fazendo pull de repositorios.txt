CONTEXTUALIZAÇÃO
Um bom desenvolvedor sabe fazer backup e dos seus dados, há situações por exemplo onde é preciso trabalhar no mesmo projeto em lugares diferentes
nesse caso podemos usar ferramentas que permitem sincronizar os códigos em vários computadores diferentes.
A ferramenta git instalada no computador permite que versões diferentes sejam salvas e atualizadas nos repositórios locais com o comando pull (puxar).

DESAFIO
Agora que você já aprendeu a fazer commits para as alterações e sincronizar os diretórios com push e pull.

Você deve usar o comando git pull origin main para sincronizar as operações.

Formar grupos de 3 conforme disposição para criar um projeto no github do relator chamado normas_abnt, todos devem enviar link do repositório.
O grupo deve fazer uma pesquisa no bloco de notas sobre as normas ABNT cada um deve fazer um pedaço em um branch e juntar as partes.

Logo os temas serão:
- Formatação: falar sobre tamanho de fontes, espaçamento entre linhas, margens, tipos de fonte etc.
- Referencias bibliográficas: mostrar na prática como devem estar as referências, de livros, de sites....
- Citações: citações são trechos de outros autores no seu texto, explique e de exemplos...

NOTA: Para fins de conhecimento os branchs devem estar no padrão parte_norma_seu_nome.
Logo serão 3 branchs:
- formatacao_seunome
- referencias_seunome
- citacoes_seunome

Ao final das atividades todos os integrantes deverão fazer um pull para sincronizar em suas máquinas e o relator do projeto deverá fazer o merge 
para o main de todas as versões (Isso será feito em sala de aula com o professor).

NOTA: Não se esqueça de fazer o push para o repositório remoto e o pull no repositório local.
