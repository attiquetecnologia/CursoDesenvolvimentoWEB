CONTEXTUALIZAÇÃO
Um bom desenvolvedor sabe fazer backup e dos seus dados, há situações por exemplo onde é preciso trabalhar no mesmo projeto em lugares diferentes
nesse caso podemos usar ferramentas que permitem sincronizar os códigos em vários computadores diferentes.
A ferramenta git instalada no computador permite que versões diferentes sejam salvas.

DESAFIO
Você é um desenvolvedor junior de sistemas e precisa criar um tutorial explicando processo de instalação do Python.
Tudo isso deverá ser feito na pasta projeto_api onde estamos desenvolvendo nossa API com Flask.

Parte 1
Você precisa criar um arquivo como o nome tutorial_python.txt (precisa ser no bloco de notas), onde você deverá fazer um commit para cada alteração.
Cada tema de commit deverá ser escrito por você desenvolvedor.

Primeiro commit deve ser: "Iniciando tutorial de Python"
Segundo commit:"Explicando o que é Python, fazendo pequena explicação de 3 linhas"
Terceiro commit: "Passo a passo fazer o download, passo 1, passo 2, passo 3"
Quarto commit: "Passo a passo de instalação"

NOTA: Os commits só funcionam se você fizer um git add nome_do_arquivo.
NOTA: Ao final precisa enviar o arquivo ao repositório com git push origin main
NOTA: Para conferir os commits use o comando git log

Parte 2
Tutorial de virtualenv

Depois de ter enviado as alterações para o servidor você precisa criar outro tutorial no mesmo modelo ensinando como configurar o virtualenv na pasta do projeto e instalar o Flask
Siga o mesmo processo para fazer o commit de cada alteração que você fez.
